PROJECT 01 View on GitHub Live Demo

Curiosity Coach: RAG Study Assistant

Grade-Aware Document Intelligence · Retrieval-Augmented Generation

Built a grade-aware RAG application that lets students query NCERT Science textbooks (Grades 6–10) using natural language, with conversational memory that resolves follow-ups like "explain that more." A single vector store serves all five grades but filters by grade at query time — grounding every answer in chapter-and-page-cited source chunks, and gracefully handling messy or ambiguous queries rather than assuming every question is clean. Ships with voice input, LaTeX math rendering, and a feedback loop that triggers answer regeneration.

PM Lens
  • →Core persona: students who need instant, textbook-accurate answers without keyword hunting — JTBD: "When I'm stuck on a Science concept, get me a grade-accurate explanation with the exact chapter it came from"
  • →Designed quality KPIs: retrieval precision, answer faithfulness, and user feedback (thumbs-up/down; regeneration capped at 2 retries) as the evaluation framework
  • →Key architecture trade-off: single multi-grade vector store with metadata filtering vs five separate stores — chose shared store for cost efficiency while preserving cross-grade awareness as a feature
  • →Designed for real-world query messiness, not just the happy path — built typo tolerance, clarifying prompts for vague questions, and off-topic redirects, since students rarely phrase questions as cleanly as a demo assumes

PROJECT 02 View on GitHub Live Demo

NEET Adaptive Learning Platform

AI-Powered Personalised Exam Prep · Multi-Agent System · EdTech

Designed and built a multi-agent adaptive quiz platform for NEET (India's medical entrance exam) covering 46 chapters and 150 concepts across Physics, Chemistry, Botany, and Zoology. Four specialised agents — Generator, Router, Evaluator, and Gap Analyser — collaborate in real time to serve questions calibrated to each student's current ability using an IRT-inspired model with Bloom's Taxonomy and Webb's DOK levels, scored with the real NEET formula (+4/−1).

PM Lens
  • →Problem: 90% of edtech prep is one-size-fits-all — north-star metric: mock exam score improvement rate; secondary: daily active practice time per student
  • →Algorithm trade-off: IRT-inspired model vs collaborative filtering — chose IRT for per-student explainability and determinism, critical for student trust in high-stakes exam prep
  • →Built "rushed-guess detection" as a data-quality signal — rapid incorrect answers scored separately from genuine knowledge gaps, preventing model drift
  • →Solved question staleness at scale — cross-session duplicate detection plus self-healing generation, where a discarded question automatically regenerates rather than leaving a gap — because repeated or near-identical questions erode student trust fast in high-stakes exam prep
